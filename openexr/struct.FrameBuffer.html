Points to and describes in-memory image data for reading and writing.

FrameBuffer does not store any image data itself, but instead points to and describes image data elsewhere in memory. Those descriptions are then used by the input and output file types to know where in memory to read from and write to when writing and reading files.

Insert a single channel.

The channel will be given the name name and when reading from a file will be filled in with the value fill if there isn't any pixel data for that channel in the file.

data is the memory for the channel and should contain precisely width * height elements, where width and height are the dimensions of the FrameBuffer.

Insert multiple channels from a slice of structs or tuples.

The number of channels to be inserted is determined by the implementation of the PixelStruct trait on T. channels should contain that number of elements, and each element is a tuple of the channel's name and default fill value.

data is the memory for the channel and should contain precisely width * height elements, where width and height are the dimensions of the FrameBuffer.
